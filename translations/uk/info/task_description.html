Перший електронний калькулятор був створений на початку 1960-х. Пристрої кишенькового розміру стали доступними у 1970-х роках, особливо після того, як Intel 4004, перший мікропроцесор, був розроблений Intel для калькулятора японської компанії Busicom. Пізніше вони часто використовувались у нафтогазовій промисловості. Дізнайся більше на сторінці Вікіпедії...

У цій серії місій ти маєш створити елементарний калькулятор.

В якості вхідних даних, ти отримаєш послідовність натискань клавіш, і, як результат твоєї функції, повинні повернути, що буде показано на екрані після натискання останньої клавіші. Будь уважний, те, що буде показано, не завжди є результатом виразу. Власне, гра з фізичним калькулятором або додатком дійсно допоможе тобі зрозуміти крайові варіанти.

Калькулятор має дуже просту клавіатуру. In the seventh mission the memory should be implemented with such operations: capital "A" (add, +), capital "S" (subtract, -), capital "G" (get value) and capital "E" (erase memory). Також дивись передумови нижче для додаткової інформації.

Як це використовується: Калькулятори широко поширені. Розуміння принципів введення та виводу даних на ньому є цікавим та корисним.

Передумови:

  • початкові операції ігноруються;
  • початкові нулі мають бути забрані, число лише з нулів - замінене на один нуль;
  • серед +- знаків між числами слід брати останній;
  • "==" означає повторення останньої операції;
  • "+=" або "-=" - додавання/віднімання числа (або результату операцій) перед комбінацією (подвоєння числа/віднімання самого від себе);
  • калькулятор ігнорує цифри, які вводяться, після 5-ї;
  • "-" для чисел < 0 НЕ займає місця цифри;
  • якщо абс.результат більше 99999 - "error" показується як результат;
  • для float, якщо цілочисельна частина абс.результату більша 9999 - "error" показується як результат;
  • для float, якщо загальна довжина числа більше 5 цифр, воно повинно бути заокруглене до 5 цифр (1.23456 -> 1.234);
  • для float, початкові і кінцеві нулі мають бути вилучені (навіть до ".", якщо можливо): 0.1200 -> .12 , 123.00 -> 123. . Це має бути зроблене після заокруглення: 1.000123 -> 1;
  • коли після B не лишається знаків, а продовжується введення цифр - приєднуй до попереднього числа: 12+В3 -> 123 ;
  • "A"/"S" increase/decrease memory by previous number with cleaning the output;
  • "G" insert memory value if used after operation, clean output and insert value othewise.
